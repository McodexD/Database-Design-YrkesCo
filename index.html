YrkesCo Database Models

Conceptual Model • Logical Model • Relationship Statements

Conceptual Model (High-Level Business View)

Shows core business entities and how they relate using boxes and lines only.

Conceptual Relationship Statements

  • A Student belongs to one Class
  • A Student has one Student Details record
  • A Class belongs to one Program
  • A Class is managed by one Education Leader
  • A Class is located at one Facility
  • An Education Leader has one Leader Details record
  • An Educator has one Educator Details record
  • A Program contains many Courses
  • Programs and Courses are connected through Program-Course

Conceptual Design Notes

  • Sensitive personal data is separated into Details entities
  • Programs and Courses are many-to-many
  • Model supports future expansion of facilities

Logical Model (Entities, Attributes, Keys)

Shows tables, primary keys, foreign keys, and structure.

Logical Relationships & Cardinality

  • Student (N) → Class (1)
  • Student (1) → Student_Details (1)
  • Class (N) → Program (1)
  • Class (N) → Facility (1)
  • Class (N) → Education_Leader (1)
  • Program (N) ↔ Course (N) via Program_Course

Primary Keys

  • student_id
  • class_id
  • program_id
  • course_id
  • leader_id
  • facility_id

Foreign Keys

  • Student.class_id
  • Class.program_id
  • Class.leader_id
  • Class.facility_id
  • Student_Details.student_id
  • Program_Course.program_id
  • Program_Course.course_id

Normalization Summary (3NF)

  • No repeating groups
  • All attributes depend on the primary key
  • No transitive dependencies
  • Sensitive data isolated in separate tables
